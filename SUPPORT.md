# Getting help with tako

## Bug reports & feature requests

Open a [GitHub issue](https://github.com/nyankobu010/tako-ai-core/issues/new/choose).
Please include:

- The version you're on (`pip show tako` and/or `cargo pkgid tako-core`).
- A minimal repro — ideally a Python snippet that runs against the
  in-process `Fake` provider so we don't need API keys to reproduce.
- Full traceback / log output (don't include real API keys; redact them).

## Questions & discussion

Use [GitHub Discussions](https://github.com/nyankobu010/tako-ai-core/discussions).
Good topics for Discussions:

- "How do I…" usage questions.
- Architecture / design feedback.
- Show-and-tell for projects built on tako.

If a discussion uncovers a bug, please file a new issue and link both ways.

## Security vulnerabilities

**Do not** open public issues or discussions for vulnerabilities. Please use
GitHub's [Private Vulnerability Reporting](https://github.com/nyankobu010/tako-ai-core/security/advisories)
on this repository — see [`SECURITY.md`](SECURITY.md) for the full policy and the
90-day coordinated-disclosure window.

## Code of Conduct

All community spaces (issues, discussions, PRs) are governed by the
[Contributor Covenant 2.1](CODE_OF_CONDUCT.md). Report violations privately
to the address in that file.
